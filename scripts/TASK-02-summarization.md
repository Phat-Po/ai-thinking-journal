# Task 02: Summarization Engine

**Goal**: Use local Ollama to compress extracted daily conversations into a structured thinking summary.

**Status**: Complete

## Input

- `output/YYYY-MM-DD-daily-thinking.md` — generated by `01_extract.py`
- Default (smart mode): ~4.5K tokens/day, well within single-call limits
- No batching needed unless user switches to `--mode full`

## Prompt Design

Example-driven format with concrete examples per section to avoid template placeholder leakage. Prompt requests bilingual section headers (Chinese + English) and auto-detects input language.

## Cost

- Local Ollama (llama3:latest): **$0.00** — fully offline, no API key needed
- Configurable via `SUMMARY_MODEL` env var

## Implementation Notes

- Script: `scripts/02_summarize.py`
- Reads `output/YYYY-MM-DD-daily-thinking.md`
- Writes `output/YYYY-MM-DD-summary.md`
- Uses local Ollama API via `urllib.request` (stdlib, zero pip deps)
- Endpoint: `http://localhost:11434/api/chat`
- Default model: `llama3:latest` (configurable via `SUMMARY_MODEL` env var)
- Timeout: 120s (sufficient for summarization)

## Usage

```bash
python3 scripts/02_summarize.py                    # summarize today
python3 scripts/02_summarize.py --date 2026-05-14  # specific date
python3 scripts/02_summarize.py --model tinyllama:latest  # different model
python3 scripts/02_summarize.py --dry-run          # preview prompt without calling Ollama
```
