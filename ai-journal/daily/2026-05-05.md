# 2026-05-05 Tuesday

## 今日主題  
今天大部分時間都在搞 prompt-randomizer 這個工具的 UI 改進，想讓常用的固定 prompt 能快速複製，還順便做了 macOS 上幾個 app cache 的盤點和清理腳本。

## 關鍵決策  
- 決定把第2步和第3步的固定 prompt 做成兩個頂部小按鈕，方便快速複製，主要是為了提升使用效率，避免每次都要手動選取。  
- 針對複製按鈕不工作的問題，改用隱藏 textarea + `document.execCommand('copy')` 作為 fallback，因為原本的剪貼簿 API 在某些瀏覽器不穩定。  
- macOS 清理腳本改用 mv 把 cache 移到垃圾桶而不是直接刪除，這樣可以避免誤刪導致資料無法恢復。  
- 清理腳本會先檢查 Arc 是否已退出，避免在程式運行中誤刪重要資料。

## 待辦事項  
- 把 prompt-randomizer 的複製按鈕功能整合到主分支，並寫個簡短說明文檔。  
- macOS cache 清理腳本測試後收集 terminal 輸出，確認 WhisperKey、Arc、Spotify、Notion 的快取清理效果。  
- 在清理前提醒用戶先確認 Arc 書籤和登入狀態，避免清理後出現問題。  
- 完成 thirst_trap_analysis_v2.jsx 的瀏覽器渲染測試，確保 React component 能正常運作。

## 思考亮點  
- 發現剪貼簿 API 在不同瀏覽器的行為差異很大，得用比較老派但兼容性好的方法才能保證複製功能穩定。  
- 用 mv 移動快取到垃圾桶比直接刪除更安全，這種可逆操作讓我更放心做清理腳本。  
- React component 的 JSX bundle 問題是因為本地環境嘗試解析 React import，標記成 external 後解決了，這提醒我打包配置要更靈活。
